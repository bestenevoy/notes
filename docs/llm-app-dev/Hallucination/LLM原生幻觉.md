# LLM 原生幻觉

## 一、什么是原生幻觉

**LLM 原生幻觉**：指**大模型仅依靠自身预训练知识、不依赖任何外部检索/工具**时，**凭空编造事实、数据、逻辑、文献、案例**的现象。

它是**模型与生俱来的缺陷**，不是工程链路问题（和 RAG、工具调用无关）。

## 二、核心本质
大模型是**概率续写模型**：
- 训练目标是：**通顺、连贯、符合语言分布**
- 不是：**100% 符合事实、真实、准确**

当模型**不确定、记忆模糊、知识空缺、上下文模糊**时，会为了保证语句流畅，**自动补全、脑补、虚构事实** → 这就是原生幻觉。

## 三、典型表现（原生幻觉专属）

1. **编造不存在论文、作者、年份、引用**
2. **编造错误公式、参数、行业数据**
3. **编造从未发生的案例、政策、历史事件**
4. **模糊知识强行笃定输出**（模棱两可的知识强行说得很绝对）
5. **逻辑自洽但事实全错**（最可怕：流畅、有理、全是假的）

## 四、原生幻觉 vs RAG幻觉 vs 工具幻觉（关键区别）

### 1. 原生幻觉

- 发生场景：**纯模型问答、无外部资源**
- 根因：模型概率生成+知识遗忘/模糊
- 特征：纯脑补、无任何外部依据

### 2. RAG幻觉

- 发生场景：**有检索文档**
- 根因：检索残缺、分块断裂、约束不足、篡改原文
- 特征：**有参考，但改错、拼接错、脑补补全**

### 3. 工具调用幻觉

- 发生场景：**需要调用工具/接口**
- 根因：偷懒不调用、编造参数、越权、格式错误
- 特征：虚构工具执行结果、伪造接口返回

## 五、原生幻觉为什么根治不了？

1. 预训练数据海量，模型**记不住所有细节**
2. 知识存在**过期、冲突、模糊**
3. 概率生成机制**优先流畅，不优先求真**
4. 模型没有“自知无知”的天然能力

## 六、唯一根治方案

**原生幻觉只能被“压制”，不能被根除**
工业级解决方式只有两种：
1. **所有事实类问题强制走 RAG**（用外部真实知识覆盖模型脑补）
2. **所有外部查询强制走工具调用**

---
### 极简一句话总结

**原生幻觉是模型天生会说谎；
RAG幻觉是有资料但读错、改错、补错；
工具幻觉是该查不查、伪造结果。**
