# 大模型训练与微调


核心：调整无序的权重到智慧过程

1. PreTrain（预训练）：填充知识
2. SFT：学会对话；LoRA：低秩矩阵微调
3. 对齐 - 培养其他能力，包含三类算法：PPO、DPO、GRPO

训练分为三大阶段：预训练→微调（SFT/LoRA）→对齐训练（PPO/DPO/GRPO）

## 阶段 1：预训练 (PreTrain)

> 基座搭建、填充知识

- 训练数据：海量互联网文本、书籍、百科、网页等通用语料，数据规模通常万亿级别 token
- 训练目标：让模型学习语言语法、世界常识、事实知识、基础逻辑，学会预测下一个字；训练完成得到基座大模型，基座知识库庞大，但不会按照人类聊天习惯回答，容易输出冗长、跑题、不合规内容
- 特点：参数量大、算力消耗极高；训练后模型拥有知识，但缺少和人类交互的能力

## 阶段 2：微调

> 包含 SFT 监督微调与 LoRA 低秩矩阵微调

### SFT

> Supervised Fine-Tuning，监督微调

- 训练数据：人工整理好的「用户提问 + 理想回答」对话样本
- 训练目标：教会模型模仿人类的对话格式，用户提问后，模型按照样本里的回答风格、结构输出内容，也就是图里的学会对话；经过 SFT 后基座模型从 “知识库” 变成能正常接话的聊天模型
- 局限：SFT 只能让模型模仿样本回答，没法区分回答好坏，容易生成看似通顺、实则质量差、不符合人类偏好的内容，因此需要后续对齐

### LoRA

> Low-Rank Adaptation，低秩矩阵微调

- 属于轻量化微调技术，是 SFT 常用的实现方式之一
- 原理：不改动模型原本庞大的权重矩阵，只新增两个很小的低秩矩阵来存储微调的权重更新；训练、推理时把小矩阵的变化叠加到原权重上
- 优势：显存需求低、训练速度快、可以针对垂直领域（比如法律、医疗）单独训练 LoRA 适配器，多个领域适配器可以灵活切换，不需要重新训练整个大模型；适合个人、中小企业做领域定制微调

## 阶段 3：对齐训练

> 培养偏好、安全、高质量输出能力，包含 PPO / DPO / GRPO

SFT 只能教会模型怎么说话，对齐是让模型的输出贴合人类偏好：优先输出有用、安全、诚实、无害的回答，拒绝有害请求，大幅提升回答质量。

### PPO

> Proximal Policy Optimization，近端策略优化

- 传统主流 RLHF（基于人类反馈的强化学习）核心算法
- 流程：先用人给不同模型回答打分，训练奖励模型；再用 PPO 微调 SFT 后的模型，让模型尽量生成能拿到高奖励分的回答；缺点是训练流程复杂、超参数难调、训练不稳定、算力开销大

### DPO

> Direct Preference Optimization，直接偏好优化

- 简化 PPO 的对齐方案，省去单独训练奖励模型、强化学习迭代的复杂流程；直接使用「同一个问题下人类优选回答、劣质回答」成对数据，直接优化模型权重，让模型倾向生成优选回答
- 优势：训练流程简单、稳定性强、算力成本低于 PPO，现在工业界大量用 DPO 替代传统 PPO 做对齐

### GRPO

> Group Relative Policy Optimization，分组相对偏好优化

- 是对 DPO/PPO 进一步优化的新一代对齐算法，DeepSeek 等模型广泛使用
- 原理：针对同一个提问让模型生成多个回答，以组内其他回答作为参照计算相对偏好，不需要大量人工标注的成对偏好数据，降低标注成本，同时提升对齐效果，小数据量也能拿到不错的对齐效果

## 训练链路

随机权重的原始模型 → PreTrain预训练（灌入海量知识，得到基座） → SFT+LoRA微调（教会模型对话格式、适配专属领域） → PPO/DPO/GRPO对齐（优化回答偏好、安全性、实用性） → 可上线使用的对话大模型


SFT（监督微调，Supervised Fine-Tuning）：是一种训练范式 / 训练目标
关注点：用「问题 + 标准答案」的对话数据，教会模型按人类习惯输出回答，解决模型会不会正常对话的问题，定义了「用什么数据、按什么目标训练」。

LoRA（低秩适配）：是一种参数更新的技术实现方式
关注点：怎么改模型权重，解决微调时要更新多少参数、耗费多少显存算力的问题，定义了「权重怎么修改」。

把微调比作练字：
- SFT = 练字的要求：照着优质字帖临摹（用成对问答样本训练，让模型学会正确回答格式）；
- LoRA = 练字的工具：不用重写整本字典（不改动模型原有全部权重），只拿便签记录笔画改动，最后贴在字典上；你也可以不用便签，直接整本重写（全参数 SFT）。
