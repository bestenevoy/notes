# LLM 工具调用幻觉：表现、成因、措施

## 一、核心表现

工具调用幻觉指模型在需要调用外部函数/接口时，脱离工具规范、伪造调用行为或虚构返回数据，分为4类典型现象：

1. **虚构工具与参数**
    不存在的工具名称、错误参数名、非法参数取值、漏传必填参数，自行编造接口字段。
2. **伪造工具返回结果（偷懒逃逸）**
    本该调用工具查询实时/外部数据，却直接靠自身知识库编造结果，完全跳过工具执行流程。
3. **越权多余调用**
    用户需求仅需少量工具，模型主动调用无关工具、批量连环无意义调用，跨权限查询他人数据。
4. **篡改/拼接工具结果**
    工具真实返回后，模型擅自修改数值、状态、文本，混合自身预训练知识覆盖接口真实数据。
5. **格式幻觉**
    输出不符合约定JSON格式，穿插自然语言，导致解析失败、调用中断。

## 二、产生原因
1. **模型底层特性**
    大模型为概率生成，优先保证语句通顺，缺少“必须使用工具获取外部信息”的天然认知；知识模糊时习惯脑补填补空白。
2. **指令约束缺失**
    System Prompt未设置刚性工具使用规则，未明确禁止编造工具、跳过调用、篡改返回；长对话上下文稀释约束，模型遗忘规范。
3. **工具管控机制缺位**
    无工具白名单、参数校验、权限分级；无重试、拦截、幂等机制，格式错乱、非法调用无阻断。
4. **上下文与流程缺陷**
    工具定义过长、历史调用记录冗余；未区分只读/高危工具，高危操作无二次鉴权；单轮无调用次数上限，引发连环越权。
5. **无后置校验兜底**
    不校验回答内容是否与工具返回内容匹配，虚构、篡改结果直接输出。

## 三、全链路解决措施
### 1. 提示词刚性约束（基础）
- 完整罗列全部工具、参数、适用场景，强制仅输出标准JSON调用格式；
- 硬性规则：外部实时数据必须调用工具，禁止编造工具/接口返回，禁止私自修改工具输出；
- 加入正反示例，明确违规会重新生成；核心规则置顶，避免上下文稀释。

### 2. 调用中间件工程拦截（核心）
1. 格式解析器：JSON解析失败自动重试，错乱输出直接驳回；
2. 参数校验器：校验参数类型、范围、必填项，非法参数打回重生成；
3. 工具白名单：拦截未定义的虚构工具名称；
4. 限流&次数限制：单轮问答最大调用次数（如3次），防止无限连环调用；
5. 幂等控制：traceId防重复调用；超时自动有限重试（最多2次）。

### 3. 权限分级，杜绝越权/危险操作
- 工具分只读、高危写操作两类，普通会话默认禁用删除、执行命令、批量导出等高风险工具；
- 高危语义识别：检测删库、文件销毁、批量隐私查询等意图直接拦截；
- 数据租户隔离：用户只能访问自身数据，禁止跨用户查询。

### 4. 生成层参数控制
温度temperature调低至0.1以下，降低创造性脑补，减少模型自行编造数据的倾向。

### 5. 后置校验兜底（最终防线）
1. 结果一致性校验：抽取回答中事实数据，与工具返回内容比对，不匹配判定幻觉，触发重生成；
2. 审计日志：记录每一次工具调用、入参、返回、违规行为，用于复盘优化；
3. 逃逸拦截：事实查询类问题强制走工具流程，禁止直接作答。
