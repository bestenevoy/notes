# Code-Review

> 代码质量管理、代码审查标准落地方案 + Vibe Coding（AI直觉式生成代码）场景专项治理

## 一、传统体系：标准代码质量管理与代码审查完整落地

### （一）代码质量管理：四层长效质量管控体系

核心思路：**左移预防+自动化门禁+持续度量+事后治理**，把质量约束嵌入开发全链路

#### 1. 事前：统一基线规范（源头控质量）

1. 编码规范标准化
    - 语言规范：ESLint/Pylint/GolangCI-Lint、格式化工具Prettier/GoFmt强制统一
    - 架构约束：分层规范、依赖禁止、模块职责、统一返回体、异常处理标准
    - 安全基线：OWASP Top10、输入校验、密钥禁止硬编码、SQL防注入、数据脱敏规则
2. 本地前置拦截（pre-commit钩子）
    提交前自动执行：格式化、Lint、简单单元测试、密钥扫描（detect-secrets），不通过禁止提交
3. 开发准入约束
    需求文档、接口契约、测试用例先行；禁止无方案直接编码。

#### 2. 事中：CI/CD自动化质量门禁（强制阻断）

流水线合并前必须全量通过，任一红灯阻断合并：

| 检查类型 | 工具 | 强制红线标准 |
| ---- | ---- | ---- |
| 静态代码分析 | SonarQube、Semgrep、CodeQL | 阻断：高危漏洞、圈复杂度>15、重复代码>20%、空指针风险 |
| 单元测试 | Jest/JaCoCo/pytest | 新增代码覆盖率≥90%，整体≥80%，无失败用例 |
| 依赖安全扫描 | Snyk、Dependabot | 禁止引入高危漏洞第三方包 |
| 构建与兼容性 | 多环境编译、API兼容性校验 | 编译报错、接口不兼容直接拦截 |
| 性能预检 | 慢SQL扫描、循环IO检测 | N+1查询、无分页全表查询标记阻断 |

#### 3. 合并：人工代码审查（Code Review）质量关卡
##### 标准流程（PR/MR闭环）
1. 开发者自审：控制单次变更≤400行，清晰描述改动、关联需求、附测试用例
2. 自动化门禁全部通过后，分配1~2名熟悉业务的评审人
3. 24小时内完成评审，区分阻断性问题（必须改）/优化建议（可选）
4. 修改后重新触发CI，二次复核通过方可合并；问题全程跟踪至闭环

##### 标准CR审查清单（5大维度）
1. **功能逻辑**：是否匹配需求、边界/异常/并发是否处理、无逻辑臆造
2. **架构可维护**：单一职责、分层不越界、无冗余、可扩展
3. **安全风险**：输入校验、权限控制、敏感数据、无注入/密钥泄露
4. **性能资源**：IO/数据库优化、资源释放、无内存泄漏
5. **工程规范**：命名、注释、测试完备、文档同步更新

#### 4. 事后：持续度量与技术债治理
- 质量看板：漏洞数量、复杂度、覆盖率、CR遗留问题、技术债时长
- 定期复盘：每周收敛阻断级漏洞，季度大规模重构高复杂度模块
- 线上反馈闭环：线上Bug反向补充CR检查清单，更新静态扫描规则

### （二）传统代码审查3种落地模式
1. **轻量线上PR评审（主流）**：GitLab/GitHub/Gerrit，异步文字评审，适合日常迭代
2. **会议集中评审**：复杂架构、核心支付/鉴权模块，多人线下逐条核对
3. **结对评审**：新人+资深开发结对编码，实时同步审查，降低漏检

## 二、Vibe Coding 定义与独有风险
### 1. 什么是Vibe Coding
Karpathy提出的AI开发模式：开发者仅用自然语言描述需求“感觉”，全权交给AI生成完整代码，人弱化理解、快速复制运行、缺少严谨设计与校验；开发者从“写代码”变成“指挥AI生成代码”，极致追求速度，弱化审查流程。
典型工具：Cursor、Claude Code、GitHub Copilot Agent、Replit AI

### 2. Vibe Coding独有的质量隐患（传统流程无法覆盖）

1. **幻觉逻辑**：AI编造不存在接口、数据库字段、业务规则，运行时才报错
2. **隐藏安全漏洞**：45% AI生成代码自带OWASP高危漏洞，且无人工校验极易上线
3. **测试缺失**：AI默认不生成完备边界/异常测试，覆盖率暴跌
4. **技术债爆炸**：一次性生成千行大文件，函数超长、耦合严重、规范混乱
5. **上下文失控**：多轮对话后AI遗忘架构约束，写出跨层、违规依赖代码
6. **无溯源**：prompt无存档，代码逻辑无人能解释，后期维护灾难
7. **密钥泄露**：prompt传入业务数据、数据库凭证，AI写入代码硬编码

## 三、Vibe Coding背景下，代码质量+审查全新落地体系
核心原则：**AI做标准化粗筛，人守住业务/安全/架构底线；全链路加AI专属约束，杜绝“凭感觉上线”**
四层防护链路：**生成前约束 → 生成中实时校验 → 生成后自动化多轮审核 → 合并前人工专项CR**

### （一）第一层：生成前源头约束（从prompt管控AI输出质量）

#### 1. 项目统一AI规则文件（强制落地）

项目根目录新增 `CLAUDE.md / AGENTS.md / VIBE_RULES.yaml`，每次AI生成自动读取约束：

- 架构规范：分层、允许依赖、禁止跨层调用
- 编码标准：Lint规则、函数行数上限、禁止any/魔法数字
- 安全红线：禁止拼接SQL、明文密钥、无权限接口
- 测试强制：生成业务代码必须同步输出单元测试+异常用例
- 输出约束：单次生成文件≤300行，复杂功能拆分多轮生成，禁止一次性生成全模块代码

#### 2. 标准化Prompt模板库（团队统一，禁止自由口述需求）

所有AI生成必须使用结构化Prompt，包含5要素：

1. 业务需求原文+需求编号
2. 项目架构、模块约束
3. 必须遵循的安全/编码规范
4. 输出要求：代码+单测+风险自查报告
5. 禁止行为：不臆造表、接口、第三方库

示例自查Prompt（内置IDE插件）：

> 生成完成后自检：1.是否处理所有边界/异常；2.有无SQL注入、硬编码密钥；3.是否符合分层架构；4.补充完整单元测试；列出全部风险点并给出修复方案

#### 3. 敏感模块AI使用禁令

支付、鉴权、加密、用户隐私、资金计算模块**禁止纯Vibe Coding**，必须先人工设计方案，AI仅辅助实现细节，全程人工主导。

### （二）第二层：生成中实时IDE拦截（即时过滤低级缺陷）

1. IDE内置实时校验（SonarLint+Linter）
AI生成代码后自动格式化、扫描代码异味、超长函数、未捕获异常，违规代码标红，不允许直接复制提交。
2. AI自审闭环（Self-Critique）
每一轮代码生成完成后，强制AI对当前diff做自查，输出风险清单并自动修复；可提升20%~30%基础缺陷检出率。
3. 上下文隔离
禁止在对话中传入真实数据库账号、密钥、生产数据；统一使用模拟测试数据。

### （三）第三层：提交后CI流水线增强（适配AI代码专项扫描）

在传统CI基础上新增**AI代码专项检测步骤**：
1. 标记AI生成代码：PR自动识别由AI产出的代码块，单独标记高风险
2. 多模型交叉复核：用另一款大模型对diff做二次安全/逻辑扫描（CodeRabbit、通义灵码）
3. 强化测试门禁：AI生成代码强制要求**新增覆盖率100%**，缺失直接阻断
4. 第三方依赖严查：AI易随机引入冷门库，新增依赖必须人工评审
5. 密钥深度扫描：git-secrets全局扫描，任何硬编码凭证直接阻断合并

### （四）第四层：合并前AI专项人工Code Review（核心防线）

传统CR清单升级，新增**Vibe Coding专属审查清单**，评审人只聚焦AI最容易出错的点，大幅提升效率：

#### 1. 必查阻断项（不通过禁止合并）

- [ ] AI是否幻觉：接口、数据表、业务规则是否真实存在，有无凭空实现逻辑
- [ ] 数据流安全：用户输入→数据库/日志全链路校验，无注入、脱敏到位
- [ ] 资源释放：数据库连接、文件、线程、缓存连接全部关闭
- [ ] 并发/事务：多线程、分布式事务有无锁失效、超事务风险
- [ ] 权限控制：接口、数据操作是否校验登录、角色权限
- [ ] 测试完备：正常、异常、边界用例全覆盖，可本地运行通过

#### 2. 架构与可维护项
- [ ] 代码分层合规，无跨层调用、领域逻辑泄露到上层
- [ ] 无超长函数（>50行）、无上千行巨型文件
- [ ] 无重复复制粘贴AI模板代码，可复用逻辑已抽取
- [ ] 新增第三方库经过团队评估，无冗余依赖

#### 3. 溯源与文档要求
- [ ] PR附带完整prompt对话记录，代码逻辑可追溯
- [ ] AI生成复杂逻辑必须补充业务注释，说明实现思路

#### 分层评审策略（降低评审压力）
1. **低风险样板代码**（CRUD、简单工具类）：自动化全绿后，1人快速复核即可
2. **中风险业务代码**（普通业务接口）：资深开发完整逐条审核
3. **高风险核心代码**（资金、鉴权、加密）：双人复核+技术负责人终审，强制签字确认

### （五）配套团队管理制度（根治Vibe Coding失控）
1. 禁止行为红线
    - 禁止AI生成代码不经任何审查直接commit/合并主分支
    - 禁止用AI重构自己未读懂的遗留系统
    - 禁止模糊一句话需求直接生成全功能代码
    - 禁止prompt携带生产密钥、真实用户隐私数据
2. 度量指标调整
    新增Vibe Coding专项指标：AI代码漏洞率、AI代码单测覆盖率、AI代码CR驳回率、幻觉缺陷线上故障数，纳入团队质量考核
3. 定期AI代码审计
    每月抽取20%AI生成线上代码做专项安全审计，汇总高频缺陷更新Prompt与静态扫描规则
4. 区分原型与生产规范
    - 原型demo：可宽松Vibe Coding，无需严格CR，但禁止合并生产分支
    - 生产代码：强制执行完整四层质量防护，无例外

## 四、落地工具栈完整推荐
### 1. 本地IDE层（Vibe Coding生成阶段）
- AI编码：Cursor、Claude Code、GitHub Copilot
- 实时校验：SonarLint、ESLint/Pylint、Prettier
- 自审插件：CodeRabbit、AI代码自查脚本

### 2. CI自动化门禁层
- 静态分析：SonarQube、Semgrep、CodeQL
- 安全依赖：Snyk、Dependabot
- 密钥扫描：detect-secrets、git-secrets
- 测试覆盖率：Codecov、JaCoCo

### 3. 代码评审平台
- GitLab MR / GitHub PR + AI评审机器人
- 专项审计：Veracode、OWASP ZAP

## 五、总结落地执行顺序
1. 先搭建**传统标准化质量+CR流程**，建立基础规范、CI门禁、评审清单；
2. 针对Vibe Coding新增前置约束文件、标准化Prompt模板；
3. 改造CI流水线，增加AI代码专项扫描、严格测试红线；
4. 升级人工CR清单，区分风险分层评审，守住业务安全底线；
5. 配套团队制度，区分原型/生产，建立AI代码质量度量与月度审计闭环。

本质：Vibe Coding是效率工具，不能替代软件工程规范；**自动化过滤AI低级错误，人类专职校验AI无法理解的业务、架构、安全逻辑**，实现人机协同的高质量交付。
